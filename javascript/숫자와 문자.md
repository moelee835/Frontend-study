### 숫자형의 바이트
https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/Number

Number는 숫자를 다루는 상수, 메서드를 가진 원시 래퍼 객체임.
Number()를 사용해서 다른 타입의 값을 숫자로 변환 가능하다.

```javascript
console.log(Number(true)); // 1

console.log(Number("1.123")); // 1.123

console.log(Number(null)); // 0

console.log(Number(undefined)); // NaN

console.log(Number("1.324Ac")); // NaN
```

Number 타입은 Java, C#등 double과 같은 배정밀도 타입의 값이다. (Sign : 1bit, Exponent : 11bits, Fraction : 52bits)

> 4바이트 단정밀도는 Sign : 1bit, Exponent : 8bits, Fraction : 23bits)

범위 바깥의 값은 `Infinity`로 대체된다.

### String 리터럴
`'` `"`  둘 중 하나로 감싸진 유니코드.
유니코드 포인트들은 Escape Sequence로 나타내야 하는 경우도 있음.
```
U+005C // \
U+000D // Carrage Return
U+000A // Line Feed
등등
```


어떤 코드 포인트..? 라도 이스케이프 시퀀스로 나타낼 수 있음. String 리터럴은 ECMAScript String 값으로 처리됨. 인코딩은 UTF-16으로 처리됨.

| --------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |

| `\\`                                                                                                                                    | backslash (U+005C REVERSE SOLIDUS)           |

#### Hexadecimal escape sequences
Escape Sequence 뿐 아니라 역 슬래쉬를 사용하여 값을 명시하는 것도 가능함.
2개의 16진수 숫자 2개를 사용 가능하다. (ASCII 범위 내에서 사용 가능한 듯 보인다.)

```javascript
'\X00FF'
'\xA9' // '@'
```


#### UNICODE ESCAPE SEQUENCES
유니코드를 사용한 이스케이프 시퀀스.
`\u` 다음 16진수 4자리로 유니코드 지정이 가능하다.

```javascript
"\u00A9"; // "@"
```


위 처럼 사용하면 4자리 제한이 있음.
4자리 이상을 명시하기 위해선 {}를 사용하여 표현함.

```javascript
"\u{2F804}";
```

> Surrogate pair?
> 2바이트 유니코드로 표현하지 못하는 경우, 2개의 2바이트 글자 쌍을 사용해서 표현함.

```javascript
"\uD87E\uDC04";
```

#### Regular Expression Literal
정규 표현식은 "/" 로 감싸서 표현한다.
Lexer(어휘분석기)가 / 를 만나면 정규식 리터럴일 수 있다고 판단한다. 이후 이스케이프 시퀀스가 아닌 /를 찾아내고 `[]` 내부에 있는 / 의 경우엔 무시하게 된다.
```javascript
/ab+c/g; /[/]/;
```

// 를 사용하면 주석으로 인지하므로 사용 불가하다. 빈 정규 표현식은 /(?:)/ 로 표현한다.
